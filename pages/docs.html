Documentation

Architecture guide, setup instructions, contribution workflow, and reference for MYCODELAB.

Overview

MYCODELAB is a static multi-page website and scientific code archive. It mirrors the structure of its GitHub repository and renders source files — Python scripts, Jupyter notebooks, Markdown, and custom HTML — inside a consistent layout framework.

The platform is built on vanilla HTML, CSS, and ES6+ JavaScript — no framework, no build step required for basic use. It is fully compatible with GitHub Pages.

ℹ

All data lives in js/data.js. Adding a project requires only a new entry in that file — no new HTML pages needed.

Installation

1. Clone the repository

bash
git clone https://github.com/sabhineet/computational-physics-library.git
cd mycodelab

2. Install Python dependencies

bash
pip install -r requirements.txt
txt — requirements.txt
numpy>=1.24
scipy>=1.11
matplotlib>=3.7
sympy>=1.12
jupyter>=1.0

3. Serve locally (optional)

Open index.html directly in any browser, or use a local server to avoid CORS issues with future fetch-based features:

bash
python -m http.server 8080
# Visit http://localhost:8080

Running Scripts

Each implementation is a standalone Python script. Navigate to the subfolder and run directly:

bash
cd codes/Root-Finding/Bisection
python Bisection_Method.py

For Jupyter notebooks:

bash
cd codes/Integration/Monte-Carlo
jupyter notebook Monte_Carlo_Integration.ipynb

Content Priority System

When the project viewer loads a project, it checks for content in this order. The first match is rendered inside the MYCODELAB layout:

PriorityFile typeRender method
1 index.html Embedded in content area via iframe-like container, global nav preserved
2 *.ipynb Notebook rendered natively — markdown, code cells, outputs, math preserved
3 README.md Markdown parsed and styled with academic typography
4 *.py Syntax-highlighted code viewer with line numbers, copy, download, GitHub link
✦

Place an index.html inside any project folder for fully custom rendered content — the MYCODELAB nav, sidebar, breadcrumbs, and theme will still wrap it.

Routing & Navigation

MYCODELAB uses query-string routing for project pages. No server-side routing is needed — everything works as a static site on GitHub Pages.

url patterns
Homepage          →  /index.html
Library           →  /pages/library.html
Category anchor   →  /pages/library.html#root-finding
Project viewer    →  /pages/project.html?cat=root-finding&id=bisection
Contributor       →  /pages/contributors.html#abhineet

Data Layer

All repository data lives in js/data.js as a plain JavaScript object attached to the global MCL namespace. This is the single source of truth for categories, projects, contributors, and metadata.

js — adding a new project
// Inside MCL.categories, find the right category and add to its projects array:
{
  id:          'false-position',
  title:       'False Position Method',
  folder:      'False-Position',
  file:        'False_Position.py',
  type:        'py',
  language:    'Python',
  author:      'agnik',           // must match a contributor id
  description: 'Regula falsi root bracketing with guaranteed convergence.',
  method:      'False Position (Regula Falsi)',
  output:      'Root: x* = 1.8393  (12 iterations)',
  tags:        ['bracketing', 'regula-falsi'],
}

Save the file — the library page, search index, sidebar, and navigation all update automatically. No HTML edits required.

Search System

The global search (⌘K / Ctrl+K) builds a flat index from all categories, projects, and contributors at runtime. Searches are performed against a concatenated text field covering: title, description, method name, language, tags, and category name.

Results are ranked by relevance and update on each keystroke. Keyboard navigation (↑↓ Enter Esc) is fully supported.

Adding Code

  1. Place your .py file (or .ipynb / README.md) inside the appropriate codes/<Category>/<Method>/ folder.
  2. Open js/data.js and add a new project object to the correct category's projects array (see Data Layer above).
  3. Commit and push. The site updates automatically.

Adding New Categories

js — new category template
{
  id:          'pdes',              // URL-safe slug
  name:        'Partial Differential Equations',
  folder:      'PDEs',              // matches codes/ subfolder name
  icon:        '∇',                 // unicode glyph
  symbol:      '∇²u=f',            // equation symbol
  color:       '#6ee7b7',           // category accent colour
  description: 'Finite difference, finite element, and spectral PDE solvers.',
  projects:    [],                  // populate as you add code
}

Push the change — the new category appears automatically in the homepage grid, sidebar, library, and search.

GitHub Pages Deployment

  1. Push the project root to a GitHub repository.
  2. Go to Settings → Pages. Set Source to Deploy from branch, branch main, folder / (root).
  3. GitHub Pages will serve index.html from the root. All relative asset paths (css/, js/, pages/) resolve correctly.
  4. The site will be live at https://<username>.github.io/<repo>/.
⚠

If the repository is not at the root (i.e. it lives at /repo-name/), ensure all href and src paths use root-relative or relative paths — avoid absolute / paths.

Code Conventions

  • Every Python file must begin with a module-level docstring describing the method, author, and references.
  • Use numpy for numerical operations; avoid heavy dependencies unless essential.
  • Include at least one worked example that produces console output matching the output field in data.js.
  • Functions must have type-annotated docstrings following NumPy style.
  • Avoid hardcoded file paths; scripts must be runnable from their own directory.

Dependencies

numpy ≥ 1.24 Array operations, linear algebra, FFT
scipy ≥ 1.11 Optimisation, integration, sparse matrices
matplotlib ≥ 3.7 Scientific plotting and visualisation
sympy ≥ 1.12 Symbolic maths for derivation verification
jupyter ≥ 1.0 Notebook execution environment

Website front-end: zero runtime dependencies. Pure HTML, CSS, ES6+ JavaScript — no npm, no bundler.
